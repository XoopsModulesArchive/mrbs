Démarrage.

  • Après avoir installé le module, aller dans les préférences du module pour défini vos valeurs et les sauvegarder.
  • Créez vos zones et salles.
  • Configurer les droits d'accès et d'administration au module (System/Groupes).
  • L'administrateur du site a automatiquement les droits complets sur le module MRBS.

Fonctionnement des thèmes sous MRBS

Les thèmes liés à MRBS sont indépendant des thèmes de votre site.

Le fichier mrbs.css.php se trouvant dans le dossier /modules/mrbs gère les valeurss communes aux thèmes.
Les fichiers de thèmes pour MRBS sont stockés dans le dossier /modules/mrbs/Themes et ont par défaut, l'exension .inc
Deux thèmes sont inclus : classic126.inc et default.inc

  • Si vous voulez un nouveau thème, créez-le, et placez-le dans ce dossier.
  • Pour pouvoir sélectionner ce thème dans les préférences du module, vous devez d'abord mettre le module à jour.

Où sont les fichiers de langue ?

MRBS a deux systèmes différents pour les fichiers de langue.

Administration du module.
Utilise la langue définie pour votre site. Tous les fichiers sont stockés dans /modules/mrbs/language/french

Partie publique du module (fonctionne de manière indépendante des préférences du site !)
Utilise les fichiers se trouvant le dossier /modules/mrbs et nommés lang.code_langue
Exemple: pour le français, le fichier de langue se nomme lang.fr

MRBS a son propre système de sélection de langue pour la partie publique, et n'utilise les préférences de votre site xoops !!
Grâce à cela, MRBS peut être multilangue.

J'ai créé un fichier de langue ou une FAQ mais je ne le vois pas dans les préférences ?

  • Le fichier se trouve-t'il bien dans le dossier /modules/mrbs ?
  • Vérifiez le nom du fichier de langue -> lang.code_langue -> Exemple lang.fr
  • Vérifiez le nom du fichier de FAQ ->  site_faq_code_langue.html -> Exemple site_faq_fr.html
  • Mettez à jour le module MRBS -> System / Modules / Mise à jour du module
  • Allez dans les préférences du module MRBS et sélectionnez votre fichier de langue ou de FAQ.
  • Enregistrer les changements.

Où puis-je changer les libellés utilisés pour les périodes et les types d'entrées ?

La partie publique de MRBS utilise son propre système de langue, indépendant de la langue de votre site. Les fichiers se nomment lang.code_langue
Exemple: le fichier de langue en français s'appelle lang.fr

Comme MRBS est multilangue, vous devez mettre à jour également tous les fichiers de langue.
Cherchez les valeurs suivantes aux alentours de la ligne 270. Vous trouverez les libellés pour les types d'entrées.

//xoops - blueteen - entry types
$vocab["A"] = "Cat1";
$vocab["B"] = "Cat2";
$vocab["C"] = "Cat3";
$vocab["D"] = "Cat4";

Cherchez les valeurs suivantes aux alentours de la ligne 300. Vous trouverez les libellés pour les périodes.

// MRBS assumes that the descriptions are valid HTML and can be output
// directly without any encoding. Please ensure that any special characters
// are encoded, eg '&' to '&amp;', '>' to '&gt;', lower case e acute to
// '&eacute;' or '&#233;', etc.
$vocab["P1"] = "Period&nbsp;1";
$vocab["P2"] = "Period&nbsp;2";
$vocab["P3"] = "Period&nbsp;3";
