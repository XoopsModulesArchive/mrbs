Getting started.

  • After installing the module go to preferences to set all configuration values and save them.
  • Create areas and rooms.
  • Make certain your xoops users / groups have accessrights to the module, System / Groups / Module Access rights
  • Website admins have full control over the MRBS module

How do themes work in MRBS?

Themes inside MRBS work independent from your site theme.

The file mrbs.css.php stored in /modules/mrbs holds the shared theme values.
MRBS themes are stored in /modules/mrbs/Themes and have by default the extension dot inc.
Two themes are included classic126.inc and default.inc

  • If you want a new theme create one and place it in this folder.
  • If you want to activate / use a new theme you must first update the module before you can select it on the preferences page.

Where are the language files?

MRBS has two different systems for the language files.

Backend system (admin section).
Uses the normal xoops language system. All files are stored in /modules/mrbs/language/yourlanguage.

Frontend system (works independent from xoops website preferences!)
Uses files located in /modules/mrbs and are called lang dot languagecode
example: french language file is called lang.fr

MRBS has its own system of choosing the frontend language and does not use the xoops website system to do this !!
Because of that your MRBS frontend can be multilanguage.

I made a language file or faq file but cannot find it in preferences?

  • Saved the file in /modules/mrbs?
  • Verify the name of the language file -> lang dot languagecode -> example lang.en
  • Verify the name of the faq file ->  site_faq_languagecode dot html -> example site_faq_en.html
  • Update your MRBS module -> Xoops System / Modules / Update
  • Go to MRBS preferences and select your language or faq file
  • Save the settings

Where do i change the words used for periods and entry type?

The frontend of MRBS uses its own independent from Xoops language system. The files are called lang dot languagecode.
Example: dutch language file is called lang.nl

Because the MRBS frontend is by default multilanguage you have to change all the language files.
Look for this in your language files around line 258. These are the words used for the entrytypes.

//xoops - blueteen - entry types
$vocab["A"] = "Cat1";
$vocab["B"] = "Cat2";
$vocab["C"] = "Cat3";
$vocab["D"] = "Cat4";

Look for this in your language files around line 285. These are the words used for the periods.

// MRBS assumes that the descriptions are valid HTML and can be output
// directly without any encoding. Please ensure that any special characters
// are encoded, eg '&' to '&amp;', '>' to '&gt;', lower case e acute to
// '&eacute;' or '&#233;', etc.
$vocab["P1"] = "Period&nbsp;1";
$vocab["P2"] = "Period&nbsp;2";
$vocab["P3"] = "Period&nbsp;3";
